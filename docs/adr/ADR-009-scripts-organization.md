# ADR-009: 스크립트 카테고리별 구조화

| 항목 | 값 |
|------|-----|
| 상태 | 승인 |
| 날짜 | 2026-03-08 |

## 문맥 (Context)

`scripts/` 디렉토리에 88개 TypeScript 파일이 평면적으로 나열되어 있었다. 파일명만으로 용도를 파악하기 어렵고, 관련 스크립트끼리 그룹핑이 안 되어 있어 유지보수가 비효율적이었다.

## 결정 (Decision)

88개 스크립트를 6개 카테고리 하위 디렉토리로 분류한다:

| 카테고리 | 용도 | 파일 수 |
|---------|------|---------|
| `check/` | 데이터 검증, 스키마 확인, 무결성 검사 | 22 |
| `upload/` | 영상/이미지 업로드 (Cloudflare Stream) | 17 |
| `data/` | 데이터 임포트/업데이트/동기화 | 25 |
| `audit/` | 감사, 분석, 리포트 생성 | 4 |
| `db/` | DB 직접 작업 (SQL 실행, 백업) | 5 |
| `tools/` | 유틸리티, 테스트, 파이프라인 | 15 |

### 규칙

- 공통 라이브러리: `scripts/lib/` (supabase.ts, cloudflare.ts 등)
- import 경로: `../lib/` (하위 디렉토리에서 상대 경로)
- `package.json` scripts는 새 경로 반영
- `scripts/README.md`에 카테고리별 설명 및 실행 예시

## 결과 (Consequences)

**장점**:
- 용도별 그룹핑으로 탐색 시간 단축
- README로 새 기여자 온보딩 용이
- 카테고리별 독립적 관리 가능

**단점**:
- 기존 문서/명령어의 경로 업데이트 필요
- 하위 디렉토리 구조로 import 경로 1단계 깊어짐

## 관련 파일
- `scripts/README.md` - 카테고리별 가이드
- `scripts/lib/` - 공통 라이브러리
- `package.json` - 스크립트 경로 (`db:backup`, `db:schema` 등)

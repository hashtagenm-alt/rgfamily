# 클라우드 계정 마이그레이션 업무 순서

> 기존 외주 개발사 계정 → 고객사 계정으로 전환
> 계정은 이미 확보된 상태 기준

---

## 마이그레이션 대상 서비스

| 서비스 | 현재 (외주 계정) | 전환 후 (고객사 계정) |
|--------|----------------|---------------------|
| GitHub | captain-yun7/rg-family | 고객사 저장소 |
| Supabase | cdiptfmagemjfmsuphaj 프로젝트 | 새 프로젝트 |

---

## 업무 순서

### 1단계: 데이터 백업

1. Supabase DB 전체 백업
   ```
   npm run db:backup
   npm run db:schema
   ```
2. Cloudflare R2 이미지 파일 전체 다운로드
3. Cloudflare Stream 영상 목록 확보 (uid, 메타데이터)
4. GitHub 저장소 전체 clone (모든 브랜치)

---

### 2단계: GitHub 저장소 이전

1. 고객사 GitHub에 새 저장소 생성
2. 기존 코드 push
   ```
   git remote add client https://github.com/[고객사]/rg-family.git
   git push client --all
   git push client --tags
   ```
3. 브랜치 보호 규칙 설정 (main)

---

### 3단계: Supabase 마이그레이션

1. 고객사 Supabase에 새 프로젝트 생성 (Region: ap-northeast-1)
2. 스키마 마이그레이션 (테이블, View, RLS, RPC 함수)
3. 데이터 복원 (백업 JSON → 새 DB)
4. 새 프로젝트 URL, anon key, service role key 확보
5. **Spend Cap ON** 설정

---

### 4단계: Cloudflare R2 마이그레이션 (이미지)

1. 고객사 Cloudflare에 R2 버킷 생성 (`rg-family-images`)
2. 백업한 이미지 파일 새 버킷에 업로드
3. 새 R2 Public URL 확보
4. API Token, Access Key 발급

---

### 5단계: 영상 스트리밍 → Vimeo 전환

1. Cloudflare Stream 영상 원본 다운로드 (또는 Google Drive 원본 활용)
2. Vimeo에 영상 업로드
3. 코드 수정:
   - 영상 재생 컴포넌트: Cloudflare Stream → Vimeo 임베드로 변경
   - 영상 업로드 API: Cloudflare Stream API → Vimeo API로 교체
   - DB: `signature_videos`, `media_content` 테이블의 cloudflare_uid → vimeo_id로 변경
4. `next.config.ts` CSP 헤더 업데이트 (Cloudflare Stream 도메인 → Vimeo 도메인)

---

### 6단계: Vercel 배포 연결

1. 고객사 Vercel에 새 프로젝트 생성
2. 고객사 GitHub 저장소 연결
3. 환경변수 등록:
   ```
   # Supabase (새 고객사 키)
   NEXT_PUBLIC_SUPABASE_URL=https://[새프로젝트].supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=[새 anon key]
   SUPABASE_SERVICE_ROLE_KEY=[새 service role key]

   # Cloudflare R2 (새 고객사 키)
   R2_ACCOUNT_ID=[새 account id]
   R2_ACCESS_KEY_ID=[새 access key]
   R2_SECRET_ACCESS_KEY=[새 secret key]
   R2_BUCKET_NAME=rg-family-images
   NEXT_PUBLIC_R2_PUBLIC_URL=[새 R2 public url]

   # Vimeo (신규)
   VIMEO_ACCESS_TOKEN=[vimeo token]

   # 기타
   NEXT_PUBLIC_USE_MOCK_DATA=false
   ```
4. 커스텀 도메인 연결 (있으면)
5. 빌드 & 배포 확인

---

### 7단계: Railway 배포

1. 고객사 Railway에 새 프로젝트 생성
2. `python-live-scraper` 배포
3. 환경변수 설정 (새 Supabase 키로):
   ```
   SUPABASE_URL=[새 URL]
   SUPABASE_SERVICE_ROLE_KEY=[새 key]
   SCRAPE_INTERVAL_SECONDS=180
   ```
4. 정상 동작 확인 (라이브 상태 업데이트 되는지)

---

### 8단계: 검증 & 마무리

1. 전체 기능 테스트:
   - [ ] 사이트 접속 정상
   - [ ] 로그인/인증 정상
   - [ ] 랭킹 데이터 표시
   - [ ] 이미지 업로드/표시 (R2)
   - [ ] 영상 재생 (Vimeo)
   - [ ] 라이브 상태 표시 (Railway 크롤러)
   - [ ] 관리자 페이지 정상
2. 기존 외주 계정 서비스 중지 (데이터 확인 후)
3. DNS 변경 (도메인 있는 경우)

---

## 주의사항

- Supabase 마이그레이션 시 RLS 정책, RPC 함수 빠뜨리지 않기
- 영상 Vimeo 전환 시 기존 Cloudflare Stream uid로 연결된 DB 데이터 전부 업데이트 필요
- 마이그레이션 중 기존 서비스는 유지하고, 새 서비스 검증 완료 후 전환
- Cloudinary 키가 .env.local에 있지만 현재 미사용 → 무시해도 됨

# VOD / Shorts 영상 업로드 구조 분석

## 개요
실제 영상 업로드를 위해 현재 VOD/Shorts 관련 코드의 구현 상태를 전체 분석함. 백엔드(DB, Server Actions, Storage)는 완성 상태이나, 사용자 대면 UI는 플레이스홀더 상태.

## 현재 구현 상태 요약

| 계층 | 상태 | 완성도 |
|------|------|--------|
| **데이터베이스** (media_content 테이블) | 완성 | 100% |
| **Server Actions** (CRUD, 조회) | 완성 | 100% |
| **관리자 UI** (/admin/media) | 완성 | 95% |
| **파일 업로드** (Supabase Storage) | 완성 | 100% |
| **사용자 표시** (메인 페이지) | 미구현 (플레이스홀더) | 0% |
| **영상 재생 UI** | 미구현 | 0% |
| **영상 상세 페이지** | 미구현 | 0% |

## media_content 테이블 스키마

```sql
CREATE TABLE public.media_content (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  content_type text CHECK (content_type IN ('shorts', 'vod')) NOT NULL,
  title text NOT NULL,
  description text,
  thumbnail_url text,
  video_url text NOT NULL,
  unit public.org_unit,        -- 'excel' | 'crew' | null
  duration int,                -- 초 단위
  view_count bigint DEFAULT 0,
  is_featured boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);
```

## 영상 업로드 방식 (2가지)

### 1. URL 입력 모드
- YouTube URL 직접 입력
- `getEmbedUrl()` 함수로 자동 embed 변환
- 지원: `youtube.com/watch`, `youtube.com/shorts`, `youtu.be`

### 2. 직접 업로드 모드 (Supabase Storage)
- VideoUpload 컴포넌트 사용
- 버킷: `videos`
- 폴더 구조: `videos/shorts/`, `videos/vod/`
- 지원 포맷: MP4, WebM, MOV, AVI
- 최대 크기: 100MB (설정 가능)
- 드래그 앤 드롭 지원

## 실제 영상 업로드 방법 (현재 가능)

### 관리자 페이지에서 업로드
1. `/admin/media` 접속 (admin 권한 필요)
2. "미디어 추가" 클릭
3. 콘텐츠 유형 선택 (Shorts / VOD)
4. URL 입력 또는 파일 직접 업로드
5. 제목, 설명, 부서, 추천 여부 설정
6. 저장

### 시드 스크립트로 일괄 등록
```bash
npx tsx scripts/seed-media.ts
```
